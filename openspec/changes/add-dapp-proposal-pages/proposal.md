# 变更：dApp 增加 Proposal 全流程页面并拆分路由

## 为什么
当前前端将空投与 Escrow 流程集中在单页，信息密度高且难以扩展；缺少投票/交付/争议的统一详情视图与事件面板，不利于用户与运营快速核验链上状态。

## 变更内容
- 将首页拆分为多页面路由：Airdrop、Proposal 列表/详情、Escrow 操作
- 新增 Proposal 详情页，展示状态机与关键字段
- 增加事件面板（投票、交付、质疑、结算、过期等）
- 保留现有功能入口并提供清晰导航

## 影响范围
- 影响规格：dapp
- 影响代码：dapp/app 路由与页面组件、dapp/lib 合约交互与事件查询
